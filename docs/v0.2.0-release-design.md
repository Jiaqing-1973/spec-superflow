# spec-superflow v0.2.0 Release Design

**Date:** 2026-06-26
**Status:** Approved
**Goal:** Bring spec-superflow to "open-source announcement ready" quality level.

## Background

spec-superflow v0.1.0 was released as an initial scaffold. Core architecture (9 skills, bridge-contract, workflow-orchestrator) and content are complete, but the project lacks:

- TypeScript compilation (no tsconfig.json, type errors in index.ts)
- End-to-end tests (no test files at all)
- Updated English documentation (Current Status still says v0.1)

The plugin is already self-contained with zero external npm dependencies. This design preserves that principle.

## Design Decisions

### 1. tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "declaration": true,
    "outDir": "dist",
    "rootDir": "src",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src/**/*.ts"]
}
```

**Rationale:** ES2022 for modern Node.js, NodeNext for ESM compatibility, declarations for type consumers, strict mode for quality.

### 2. Fix index.ts Duplicate Types

`src/index.ts` exports `Scenario` and `Requirement` from `./schema/base.js` at the top, then re-declares empty versions of those interfaces at the bottom. Delete the duplicate declarations (lines with `export interface Scenario { rawText: string; }` and `export interface Requirement { text: string; scenarios: Scenario[]; }`).

### 3. package.json Updates

```json
{
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "files": ["dist", "skills", "hooks", "scripts", "templates", "docs"],
  "scripts": {
    "build": "tsc",
    "test": "node --test tests/",
    "validate": "node scripts/validate-artifacts"
  }
}
```

### 4. Integration Tests

**File:** `tests/e2e.test.ts`
**Runner:** Node.js built-in `node:test` module (zero dependencies)
**Strategy:** Use real example artifacts from `docs/examples/`

Test cases:
1. `parseDeltaSpec` on `add-dark-mode/specs/ui-theme.md` → expect 3 ADDED requirements
2. `parseDeltaSpec` on `refactor-auth-boundary/specs/auth-boundary.md` → expect 3 MODIFIED requirements
3. `parseChangeMarkdown` on `add-dark-mode/proposal.md` → expect valid ParsedChange
4. `parseChangeMarkdown` on `refactor-auth-boundary/proposal.md` → expect valid ParsedChange
5. `Validator.validateSpecContent` on both specs → expect valid: true
6. `Validator.validateChangeContent` on both proposals → expect valid: true

This validates:
- Parsing correctness against real data
- Validator behavior on well-formed content
- Example files remain correct (documentation-as-test)

## Non-Goals (Deferred to v0.3.0)

- P2 items: skill usage guides, performance benchmarks, CI/CD
- npm package publishing (may revisit based on community interest)
- Additional test coverage (unit tests for individual parsing/validation functions)

## Key Principles Preserved

- ✅ Zero external npm dependencies
- ✅ Pure TypeScript interfaces (no runtime validation)
- ✅ Regex-based parsing (no Zod)
- ✅ Self-contained (no OpenSpec/Superpowers runtime)
- ✅ De-duplicated skill set (brainstorming→spec-explorer, writing-plans→spec-forger, etc.)
- ✅ bridge-contract as the unique original layer
